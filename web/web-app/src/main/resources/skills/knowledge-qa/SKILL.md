# 知识问答技能

## 使用场景

- 用户询问平台规则、预约流程、签约流程、订单超时、入住须知、押金支付、常见 FAQ
- 用户需要基于知识库获取说明性答案

## 业务规则

- 说明性问题优先调用 `searchKnowledge`
- `searchKnowledge` 适合回答规则、流程、FAQ，不适合查实时预约列表或实时订单列表
- 如果用户问题本质上是实时业务查询或执行动作，要切到对应业务技能，而不是继续用知识库兜底
- 当知识库返回内容不足时，要明确说明“当前知识库未检索到足够信息”，不要编造规则

## 工具调用建议

- 平台规则、FAQ、流程说明：`searchKnowledge`
- 用户问当前预约：切到预约服务技能并调用 `listMyAppointments`
- 用户问当前订单：切到签约订单技能并调用 `listMyLeaseOrders` 或 `getLeaseOrderDetail`
- 用户问房源列表：切到房源查询技能并调用 `searchRooms`

## 回复要求

- 先给结论，再给简要依据
- 引用知识库时尽量保持简洁，不要长篇复述
